# 📘 정보 구조 설계서 (Information Architecture)

본 문서는 익명 기반 온라인 셀러 커뮤니티 웹서비스의 정보 구조(IA: Information Architecture)를 정리한 문서입니다.  
이 문서는 UX 향상, 서비스 구성 명확화, 개발 가이드를 위해 사용되며, 반응형 웹 및 SEO를 고려하여 작성되었습니다.

---

## 🗺️ 1. 사이트 맵 (Site Map)

```text
/
├── 소개 (intro)
├── 글 목록 (posts)
│   ├── 전체글 (all)
│   ├── 베스트글 (best)
│   ├── 최신글 (recent)
│   └── 카테고리별 (category/[slug])
├── 검색결과 (/search?q=)
├── 글 상세 (/post/[id])
├── 글 작성 (/write)
├── 로그인 (/login)
├── 회원가입 (/signup)
└── 내 정보 (/me) – 로그인 유저만
---

## 2. 🔁 User Flow

### ✅ 회원
- 홈페이지 접속 → 글 탐색 → 글 읽기/검색 → 글 작성/댓글/좋아요
- 마이페이지 접속 → 내가 쓴 글 목록 열람

### 🚫 비회원
- 홈페이지 접속 → 글 탐색 및 읽기 가능
- 글 작성/댓글/좋아요 시 로그인 유도

---

## 3. 🧭 Navigation Structure (Topbar)

| 메뉴 | 설명 |
|------|------|
| 로고 | 홈으로 이동 |
| 소개 | 서비스 목적/가이드 |
| 글 메뉴 | 전체글, 베스트, 최신, 카테고리별 |
| 검색창 | 키워드 검색 (전체 글 대상) |
| 로그인/회원가입 버튼 | 인증이 필요한 경우 표시 |
| 마이페이지 버튼 | 로그인 시 노출 (내 글 확인 가능) |

※ 모바일에서는 햄버거 메뉴로 축소, 검색창은 상단 고정 유지

---

## 4. 🏗 Page Hierarchy

| 레벨 | 페이지 | 설명 |
|------|--------|------|
| 1단계 | `/` | 홈 (서비스 소개 + 주요 글 요약) |
| 2단계 | `/posts` | 글 리스트 |
| 3단계 | `/post/[id]` | 글 상세 |
| 3단계 | `/write` | 글 작성 (로그인 필요) |
| 1단계 | `/login`, `/signup` | 인증 페이지 |
| 1단계 | `/me` | 마이페이지 (내 글 확인) |

---

## 5. 📚 Content Organization

| 콘텐츠 유형 | 구성 요소 |
|-------------|-----------|
| 글(Post) | 제목, 본문, 카테고리, 작성일, 조회수, 댓글 수, 공감 수 |
| 댓글(Comment) | 본문, 작성일, 공감 수, 신고 기능 |
| 카테고리 | 플랫폼 유형(예: 스마트스토어, 쿠팡 등) |
| 사용자 정보 | 익명번호 또는 닉네임, 작성 글 |
| 검색 결과 | 키워드 기반 글 리스트 출력 |

---

## 6. 🤝 Interaction Patterns

| 인터랙션 | 결과 |
|----------|------|
| 글 작성 | 로그인 필요, 작성 완료 후 글 상세로 이동 |
| 댓글 작성 | 로그인 필요, 실시간 반영 |
| 공감 클릭 | 로그인 필요, 실시간 증가, 중복 방지 |
| 글/댓글 신고 | 신고 사유 선택 → 처리 안내 팝업 표시 |
| 검색 | 검색어 입력 시 관련 게시글 실시간 표시 |
| 반응형 | 모바일 기준: 카드 UI, 드롭다운, 터치 최적화 |

---

## 7. 🌐 URL Structure

| 페이지 | URL 예시 | 설명 |
|--------|----------|------|
| 홈 | `/` | 소개 및 주요 콘텐츠 |
| 전체글 | `/posts` | 전체 글 리스트 |
| 베스트 | `/posts/best` | 인기 글 기준 정렬 |
| 최신 | `/posts/recent` | 최신 순 정렬 |
| 카테고리 | `/posts/category/smartstore` | 특정 플랫폼 필터 |
| 글 상세 | `/post/123` | 개별 글 열람 |
| 글 작성 | `/write` | 글 작성 폼 |
| 검색 | `/search?q=택배지연` | 키워드 기반 검색 |
| 로그인 | `/login` | 로그인 폼 |
| 회원가입 | `/signup` | 회원가입 폼 |
| 마이페이지 | `/me` | 내 정보 및 글 확인 |

---

## 8. 🧱 Component Hierarchy

### 📌 공통 컴포넌트
- `Topbar`: 로고, 메뉴, 검색창, 인증 버튼
- `Footer`: 간단한 안내 및 약관 링크

### 📝 글 리스트 페이지
- `PostList`: 글 목록
  - `PostCard`: 썸네일, 카테고리, 공감수, 댓글수 등 요약 정보

### 📄 글 상세 페이지
- `PostDetail`: 본문, 작성일, 조회수, 공감 버튼
- `CommentSection`: 댓글 리스트
  - `CommentItem`: 댓글 본문, 공감, 신고

### ✍ 글 작성 페이지
- `PostEditor`: 제목 입력, 본문 에디터, 카테고리 선택, 작성 버튼

### 🔐 인증 관련
- `AuthForm`: 이메일/비밀번호 입력, 제출

### 👤 마이페이지
- `UserProfile`: 사용자 정보
- `MyPosts`: 내가 쓴 글 목록

---

✅ **접근성과 UX 고려 사항**
- 키보드 내비게이션, 명확한 버튼 레이블, 스크린 리더 지원
- 모바일 환경에서 터치 영역 확보 및 콘텐츠 단순화
- 시각 대비와 폰트 가독성 준수

✅ **SEO 최적화 요소**
- 각 페이지마다 title, meta description, og 태그 포함
- slug 형태의 URL 설계 고려 (예: `/post/123-리뷰테러-어떡하죠`)
- 크롤러 인덱싱을 위한 robots.txt 설정 및 sitemap.xml 자동 생성
